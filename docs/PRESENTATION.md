# 校园轻集市 — 项目展示提纲

> 预计时长：3—5 分钟
> 展示重点：业务功能、技术实现、开发过程、AI 协作

---

## 1. 项目背景（30 秒）

**"校园轻集市"** 是一个面向高校校园生活场景的 PC 端 Web App。

校园里每天都有大量零散需求：
- 毕业季的二手物品流转
- 食堂/图书馆遗失物品的招领
- 奶茶、外卖的拼单凑满减
- 代取快递、代打印等跑腿委托

这些需求分散在 QQ 群、微信群、表白墙，信息杂乱、难以检索。本项目将这些场景整合到一个平台上，提供统一的浏览、发布、搜索和收藏体验。

---

## 2. 技术栈（20 秒）

| 技术 | 用途 |
|------|------|
| Vue 3（Composition API） | 前端框架 |
| TypeScript | 类型系统 |
| Vite | 构建工具 |
| Vue Router | 路由管理 |
| Pinia | 状态管理 |
| Axios | HTTP 请求 |
| JSON Server | Mock 后端 |
| ECharts | 数据可视化 |

---

## 3. 页面结构与路由（30 秒）

项目共 **13 个页面、12 条路由**，按业务分为四个模块：

```
首页 (/)
├── 二手交易 (/trade)      ← 核心业务
├── 失物招领 (/lost-found)
├── 拼单搭子 (/group-buy)
├── 跑腿委托 (/errand)
├── 市场总列表 (/list)     ← 统一浏览入口
├── 详情页 (/detail/:id)
├── 发布页 (/publish)      ← 数据入口
├── 用户中心 (/user)       ← 个人数据管理
├── 消息中心 (/message)
├── 数据看板 (/dashboard)  ← ECharts 可视化
├── 登录 (/login)
└── 注册 (/register)
```

演示时展示：**首页 → 点击导航 → 进入二手交易 → 查看列表 → 点击卡片 → 查看详情 → 返回**

---

## 4. 核心功能演示（90 秒）

### 4.1 列表浏览与搜索

1. 进入 **二手交易** 页面，展示列表数据从 JSON Server 加载
2. 在搜索框输入关键词，展示筛选结果
3. 切换到 **失物招领**、**拼单搭子**、**跑腿委托**，展示不同类型列表

### 4.2 收藏功能

1. 在二手交易列表中点击 ❤️ 收藏一条信息
2. 进入 **用户中心**，展示"我的收藏"列表中有刚才收藏的内容
3. 返回列表，取消收藏，展示状态同步更新

### 4.3 发布功能

1. 进入 **发布页**，选择一个分类（如"二手交易"）
2. 填写标题、价格、描述、校区等信息
3. 点击提交，展示跳转到对应列表页
4. 在列表中展示新发布的数据

### 4.4 状态交互

1. **加载状态**：刷新页面时展示 LoadingState 组件
2. **空状态**：搜索不存在的内容时展示 EmptyState 组件
3. **错误状态**：停止 Mock 服务后刷新，展示 ErrorState 组件

## 5. AI 协作说明（30 秒）

本项目使用 AI Coding 工具辅助开发，AI 参与了以下阶段：

| 阶段 | AI 做了什么 | 我做了什么 |
|------|------------|-----------|
| 页面骨架 | 生成 6 个页面 + 路由配置 | 检查页面命名、路由路径 |
| Mock 数据 | 生成 JSON Server 数据结构 | 调整字段贴合校园业务 |
| 发布表单 | 生成表单组件和校验逻辑 | 删除图片上传等超出范围的功能 |
| 状态管理 | 生成 Pinia Store 代码 | 修复类型错误、调整状态边界 |
| 交互优化 | 生成状态组件模板 | 筛选优化建议、避免过度复杂 |

**核心体会**：AI 提高了重复性工作的效率，但业务理解、类型修复、功能取舍和质量判断**必须由开发者自己完成**。

## 6. 遇到的问题与解决（30 秒）

| 问题 | 解决方式 |
|------|----------|
| TypeScript 类型错误（`string\|number` → `number`） | 添加 `Number()` 显式转换 |
| AI 生成了图片上传、支付等超出范围功能 | 逐项审查并删除 |
| Mock 数据字段不贴合校园业务 | 补充 `campus`、`status`、`deadline` 等字段 |
| 组件过度抽象导致维护困难 | 合并简化嵌套组件 |
| JSON Server 数据同步问题 | 统一 API 调用方式 |

---

## 7. 项目不足与改进方向（20 秒）

| 当前局限 | 改进方向 |
|----------|----------|
| JSON Server 模拟后端 | 接入 Node.js / Spring Boot 真实后端 |
| 模拟登录，无 JWT 鉴权 | 接入真实认证服务 |
| 图片仅支持 URL 输入 | 集成文件上传 |
| 搜索为前端筛选 | 接入后端全文检索 |
| Dashboard 包体积大（ECharts） | 动态导入按需加载 |
| 移动端适配不足 | 增加响应式设计 |
